# Screenshot Directory

Place your CerebrumOS screenshots in this directory with the following names:

## Required Images

1. `playground.png` - Runtime Playground main dashboard
2. `pipeline.png` - Active request pipeline visualization
3. `workers-memory.png` - Thread pool and memory metrics
4. `timeline.png` - Timeline Replay interface
5. `job-details.png` - Job lifecycle details (8 stages)
6. `decision-engine.png` - Decision Engine explainability UI
7. `memory-viz.png` - Memory visualization with heat map
8. `cache-dashboard.png` - Radix Cache dashboard
9. `aitop.png` - AI-Top system telemetry
10. `benchmarks.png` - Benchmark comparison charts
11. `architecture.png` - Interactive system architecture
12. `algorithms.png` - Algorithm visualizer
13. `historical-metrics.png` - Historical benchmarking trends
14. `settings.png` - Cluster settings UI
15. `logs.png` - System logs viewer

## Image Guidelines

- **Format**: PNG (recommended), JPG, or WebP
- **Resolution**: 1920x1080 or higher
- **File Size**: < 2MB per image
- **Quality**: High quality, clear UI elements
- **Dark Theme**: Screenshots should use the dark theme for consistency

## Capturing Screenshots

1. Run the application: `npm run dev` (frontend) and `python main.py` (backend)
2. Navigate to http://localhost:3000
3. For each page, capture full-screen screenshot
4. Save with the exact filename from the list above
5. Place all images in this directory

## Updating README

After adding images, the main README.md will automatically display them in the Screenshots section.
